# Dazong 演示控制台

独立的 Vue 3 开发工具，用于在隔离环境读取真实基础资料、批量创建订单、推进演示履约和清理订单链路。

它不是生产运营后台，后端必须启用 `DEMO_MODE=true` 才能使用相关能力。

## 启动

推荐从主仓库启动：

```bash
make dev-up
```

访问：

```text
http://127.0.0.1/demo-console/
```

独立开发：

```bash
cd demo-console
npm install
npm run dev
```

默认开发端口为 `15175`。开发代理目标可通过 `VITE_API_PROXY` 配置。

## 数据来源

- 采购方和食堂：`GET /api/demo/client-accounts`
- 供货方：`GET /api/demo/supplier-accounts`
- 商品：真实商品搜索接口
- 合约：真实合约与下单校验
- 下单：`POST /api/orders`

采购方列表按启用食堂展开。下单前控制台会使用采购方账号登录，再绑定选中的食堂。

接口失败时控制台不会填充内置假账号或假商品。

## 主要能力

### 造数

- 批量创建不同商品、数量、日期和时段的订单。
- 校验食堂、合约、商品分类和配送商。
- 为智能分单、排线、PDA、司机和监管页面准备数据。

### 推进履约

- 提交或准备分单。
- 批量标记供货分单已出库。
- 记录演示标签打印审计。
- 推进取货、送达、收货和结算。
- 准备调度车次演示。

### 清理

- 删除指定订单及其履约关联数据。
- 清空全部订单链路。
- 清理调度、分检、收货、退货、账务、工单、通知、告警、审计、幂等键和 Outbox 等关联记录。

清理不会删除账号、食堂、商品、分类、合约、车辆、设备和仓库等基础资料。

## 演示账号

本地种子环境通常提供：

```text
采购方：client001 至 client006
配送商：delivery001
监管方：monitor001
供货方：supplier001 等
统一演示密码：demo123
```

这些账号只允许用于隔离演示环境，不能用于生产。

## 安全

- 生产必须设置 `DEMO_MODE=false`。
- 不要把临时隧道地址写入源码或文档。
- 删除操作必须核对 API 根地址并完成页面二次确认。

详细步骤见 [演示控制台操作手册](../docs/操作手册/10-演示控制台.md)。
