# 系统总览

## 1. 九端如何组成

系统有六类 PC 业务角色：采购方、供货方、配送商、指定厂家、运营方和监管方；另有智能秤、分检 PDA、司机端三个 Android 终端。

演示控制台、天枢、BI 和识别训练是辅助工具或功能模块，不是额外账号角色。

## 2. 主业务链路

```text
招标与授标
  -> 采购方按食堂下单
  -> 配送商智能分单
  -> 供货方/指定厂家出库
  -> PDA 分检
  -> 路线排程与装车
  -> 调度发车
  -> 司机送达
  -> 智能秤或采购方收货
  -> 退货/质检/投诉
  -> 对账与结算
  -> 监管分析
```

## 3. 订单状态

```text
下单 -> 配货 -> 发货 -> 收货 -> 收货确认 -> 已结算
  \-> 取消
```

配送页面会显示更细的操作阶段，例如“待分单、待供货商发货、待分拣、待取货、配送中、待客户收货、待结算”。这些阶段由订单、分单和扫码记录计算得出，不是第二套订单状态。

## 4. 登录规则

- 六类 PC 角色从 `/login` 登录。
- 采购方登录后必须在 `/client/select-canteen` 选择食堂。
- 智能秤使用采购方账号登录，也必须选择食堂。
- 分检 PDA 使用配送商账号登录。
- 司机端使用车辆车牌登录，令牌与六角色令牌分离。

若页面提示无权限，先确认账号角色、账号状态和采购方食堂上下文。

## 5. 通知与追踪

- 各业务端通过消息中心查看订单、工单、账务和监管广播。
- 订单状态和关键动作写入状态日志或审计日志。
- 监管端可查看订单稽核链和跨角色事件。
- 页面数据以服务端记录为准，不以演示截图或宣传材料为准。
