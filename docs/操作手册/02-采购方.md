# 采购方

入口：`/client/dashboard`

## 1. 选择食堂

采购方账号可绑定多个食堂。登录后先进入 `/client/select-canteen`：

1. 选择一个启用食堂。
2. 系统换取带 `canteen_id` 的新令牌。
3. 进入采购方首页。

商品、合约、订单、账单和收货都按当前食堂隔离。切换食堂后应重新加载页面数据。

## 2. 招标与合约

- `/client/contracts/new` 创建招标。
- `/client/tenders` 查看招标过程。
- `/client/contracts/{id}/bids` 查看投标和评标上下文。
- `/client/contracts` 查看已形成的合约。

下单日期和分类必须被有效的“已中标”合约覆盖。

## 3. 下单

入口：`/client/orders/new`

1. 选择期望配送日期和时段。
2. 搜索合约范围内商品。
3. 填写数量并检查单位、参考价格和预计金额。
4. 可使用 OCR 图片或语音辅助生成订单行。
5. 检查识别结果，修正未匹配、重复或数量异常的行。
6. 提交订单。

OCR 和语音是录入辅助，不会绕过商品、合约、食堂和幂等校验。

## 4. 查看订单

- `/client/orders` 查看订单列表。
- `/client/orders/{id}` 查看履约和物流。
- 待收货订单可进入 `/client/receive/{id}`。

订单取消只在后端允许的早期状态执行。已进入后续履约的订单应通过差异、退货或投诉处理。

## 5. 收货

网页端和智能秤都使用同一套订单收货数据：

1. 核对商品和应收数量。
2. 逐行填写或锁定实收值。
3. 少收或退货时填写原因和数量。
4. 根据现场要求上传图片。
5. 完成签字和整单确认。

订单送达后主状态为“收货”；采购方确认后进入“收货确认”。

## 6. 投诉与账单

- 在订单详情针对异常发起投诉并上传图片。
- `/client/bills` 查看当前食堂账单和结算状态。
- 投诉不会自动取消订单或账单，应等待配送商回复和运营结案。
