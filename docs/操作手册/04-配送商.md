# 配送商

入口：`/delivery/workbench`

## 1. 工作台阶段

工作台按真实履约数据显示：

- 待分单。
- 待供货商发货。
- 待分拣。
- 待取货。
- 配送中。
- 待客户收货。
- 待结算。

这些阶段由订单、分单和 PDA 扫码记录派生。

## 2. 供应商与报价

- `/delivery/suppliers` 管理挂靠本配送商的供货方。
- 查看供货方商品报价和分类覆盖。
- 指定厂家由商品配置决定，不按普通供货方选择。

智能分单前应确保普通商品至少有一个有效供货报价。

## 3. 智能分单

入口：`/delivery/smart-split`

1. 选择待分单订单。
2. 运行分单预览。
3. 检查推荐供货方、报价、指定厂家约束和不可分配原因。
4. 必要时调整。
5. 提交分单。

预览不落库；只有提交成功才形成正式 `OrderItemAllocation`。

## 4. PDA 分检

供货方出库后，配送中心使用 PDA 扫描分单标签。工作台会等待全部分单完成扫码。

出现“未出库、非当日订单、不属于当前配送商、重复扫描”等提示时，应先修正上游状态，不能直接跳过。

## 5. 智能排线与车次

入口：`/delivery/smart-routing`

1. 选择计划日期和待配送订单。
2. 检查地址、经纬度、时窗、服务时长、重量和体积。
3. 生成路线建议。
4. 检查车辆容量、限行、距离和预计时间。
5. 提交路线，生成调度车次。

入口：`/delivery/dispatch-trips`

1. 查看车次站点和随车商品。
2. 确认商品是否完成出库和分检。
3. 逐项装车。
4. 正常发车，或在有阻塞时记录原因和照片后异常发车。
5. 必要时追加站点、编辑车次或取消未发车车次。

## 6. 车辆、设备与仓库

- `/delivery/vehicles`：车辆、司机、容量和状态。
- `/delivery/devices`：北斗、摄像头等设备。
- `/delivery/warehouses`：仓库、摄像头和精创冷云设备。

设备绑定后才能在履约和监管页面展示对应位置、直播或温湿度数据。

## 7. 送达、差异与投诉

- 司机送达后订单进入“收货”。
- `/delivery/receiving-differences` 查看少收和退货。
- 启用退货复核的配送商需要审核退货记录。
- `/delivery/complaints` 回复采购方投诉，运营方最终结案。

## 8. 账单

入口：`/delivery/bills`

配送商同时可能存在采购方应收、供货方应付和指定厂家应付。结算前应核对食堂、订单、收货差异和对账状态。
