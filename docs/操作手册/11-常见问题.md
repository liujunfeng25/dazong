# 常见问题

## 为什么采购方登录后还要选食堂？

一个采购账号可以管理多个食堂。订单、合约、账单和收货按食堂隔离，因此必须换取带食堂上下文的令牌。

## 为什么采购方看不到商品？

依次检查：

1. 是否选择食堂。
2. 是否存在覆盖配送日期的已中标合约。
3. 商品分类是否在合约范围内。
4. 商品是否启用且未删除。

## 为什么供货方看不到订单？

供货方只看到已分配给自己的订单行。检查配送商是否已经提交智能分单，以及账号是否挂靠正确的配送商。

## 为什么智能分单没有候选供货方？

普通商品需要挂靠当前配送商的供货方报价；指定厂家商品需要商品配置正确的指定厂家。

## 为什么 PDA 提示“供货商尚未出库”？

标签对应分单还没有完成出库。先让供货方或指定厂家完成标签和出库登记。

## 为什么车次不能发车？

常见原因是商品未出库、未分检或未装车。应处理阻塞；确需发车时由调度员记录原因和照片后执行异常发车。

## 为什么司机端看不到车次？

检查车牌、车辆状态、车次计划日期、车辆分配和车次状态。司机端只显示属于当前车辆的有效车次。

## 为什么司机端不能开始配送？

司机端不能发车。调度端必须先执行整车发车或异常发车，使车次进入“运输中”。

## 为什么智能秤没有订单？

检查 API 地址、采购方账号、食堂选择以及订单是否已经由司机送达并进入待收货状态。

## 为什么地图或 AI 会超时？

VPN、代理、DNS、出口白名单、供应商配额或国内外网络路径都可能造成超时。先确认本地 API 健康，再检查对应外部服务。降级结果不等于外部服务真实成功。

## 工厂端为什么没有加工任务？

当前工厂端是指定厂家履约端，只处理指定商品的订单行、质检和账单。系统没有加工生产工单。

## 完整接口在哪里看？

后端启动后查看 `/docs`；主前端也提供 `/docs` 接口说明。操作手册不维护完整接口副本。
