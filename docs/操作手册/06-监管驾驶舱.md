# 监管驾驶舱

入口：`/monitor/dashboard`

## 1. 监管范围

监管端提供跨主体只读态势和监管动作，不代替采购、供货、配送或运营执行日常业务。

主要入口：

- `/monitor/dashboard`：核心态势、履约监控、指挥广播、数据挖掘和 AI。
- `/monitor/tianshu`：天枢大屏。
- `/monitor/price-cockpit`：价格驾驶舱。
- `/monitor/route-planning`：路线规划展示。
- `/monitor/orders`：订单监管。
- `/monitor/logistics`：物流监管。
- `/monitor/alerts`：告警。
- `/monitor/reports`：监管报告。

## 2. 核心态势

核心态势聚合订单生命周期、GMV、履约、异常、质检、账务和事件流。空数据应显示零值或空态，不代表接口失败。

打开订单稽核详情时，可查看订单、分单、状态、收货、退货、质检、账务和审计日志。

## 3. 履约监控

履约监控展示：

- 调度车次和站点。
- 车辆北斗位置和轨迹。
- 摄像头直播。
- 仓库冷链设备。
- 超时、阻塞、温湿度和配送异常。

设备数据缺失时先检查绑定、第三方配置、网络和设备在线状态。

## 4. 指挥广播

监管广播支持：

1. 选择业务全域、角色或指定用户。
2. 填写标题、内容和优先级。
3. 发送后查看接收人数。
4. 查看接收人和已读时间。

广播会进入目标角色的消息中心。

## 5. 价格与数据挖掘

- 新发地模块用于北京行情、历史趋势和预测训练。
- 农产品价格网模块用于全国价格、地区对比、价差、质量校验和预测。
- 抓取、补抓和训练是异步或长耗时操作，应查看进度和日志。
- 网络失败时使用已有数据，不得把旧数据描述为刚刚抓取成功。

## 6. AI 助手

AI 助手分两类问题：

- “当前有多少订单、哪些车异常、价格如何”：调用实时业务工具。
- “在哪里操作、怎么分单、司机怎么送达”：检索本操作手册。

模型不可用时系统可以降级到本地工具和规则。回答应明确说明降级，不代表 DashScope 已成功调用。

## 7. 告警

入口：`/monitor/alerts`

告警状态为 `open` 或 `closed`。关闭告警前应确认关联订单、车辆、设备或账务问题已处理。
