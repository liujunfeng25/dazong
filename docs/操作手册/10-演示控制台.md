# 演示控制台

入口：`/demo-console/`

> 内部工具。只能在 `DEMO_MODE=true` 的隔离开发或演示环境使用。

## 1. 用途

- 读取真实采购方、食堂、供货方、商品和合约。
- 批量创建演示订单。
- 模拟供货方出库、取货、送达、收货和结算。
- 准备调度演示数据。
- 删除指定订单链路或清空全部订单链路。

它不是生产运营后台。

## 2. 演示账号

默认种子环境使用：

```text
采购方：client001 至 client006
配送商：delivery001
监管方：monitor001
供货方：supplier001 等种子账号
统一演示密码：demo123
```

账号只适用于本地种子环境。正式环境不得沿用这些密码。

## 3. 批量下单

1. 刷新采购方列表。
2. 选择“采购账号::食堂”。
3. 设置订单数量、商品行数、数量范围、配送日期和时段。
4. 检查有效合约和商品池。
5. 创建订单。

控制台走真实下单接口，因此仍受食堂、合约、商品和幂等规则限制。

## 4. 推进履约

可按演示顺序执行：

1. 智能分单。
2. 供货方批量出库。
3. PDA 分检或演示分检准备。
4. 路线提交和车次准备。
5. 取货、送达、收货。
6. 结算。

演示跳过真实打印等动作时，页面和审计记录应明确标识为演示操作。

## 5. 清理

- 删除指定订单会连同其履约、车次、收货、账单、工单、通知和审计关联数据清理。
- 清空全部订单链路是破坏性操作。
- 清理不会删除账号、食堂、商品、分类、合约、车辆和设备等基础资料。

执行前确认当前连接的是隔离环境，并按页面要求输入二次确认文本。
