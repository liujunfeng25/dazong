# 运营端

入口：`/operation/dashboard`

## 1. 运营端职责

- 维护分类、商品、账号和采购方食堂。
- 查看合约和订单全貌。
- 处理收货差异、投诉和其他工单。
- 配置全局或定向账期，查看账单与对账。
- 管理智能秤图像识别类别、样本、ROI、训练和发布。
- 查看车辆、仓库、摄像头和冷链设备的运营视图。

## 2. 基础资料

### 分类与商品

1. 进入 `/operation/categories` 维护一级、二级分类。
2. 进入 `/operation/products` 新增或编辑商品。
3. 指定厂家商品需要开启指定厂家标记并选择厂家账号。
4. 根据质检要求选择批次报告或周期报告。
5. 重量、尺寸和体积参数会影响排线容量计算，应使用真实单位。

不要直接删除已被业务数据引用的基础资料；页面会根据后端校验提示改为停用或阻止删除。

### 账号与食堂

1. 在 `/operation/accounts` 创建六类业务账号。
2. 普通供货方需要绑定归属配送商。
3. 在 `/operation/client-canteens` 为采购方创建食堂。
4. 核对食堂地址、经纬度和状态，否则下单和路线规划可能缺少位置。

## 3. 合约与订单

- `/operation/contracts` 查看招投标结果和有效期。
- `/operation/orders` 查看全量订单。
- `/operation/orders/{id}` 查看订单行、分单、收货、质检、退货和日志。
- `/operation/receiving-differences` 查看少收、退货及其他收货差异。

运营端主要负责监督和纠偏，不代替供货、调度、司机和采购方执行正常履约动作。

## 4. 工单

入口：`/operation/tickets`

工单类型包括异常订单、售后投诉、配送异常和账务异常。投诉通常由采购方发起、配送商回复、运营方结案。

工单状态：

```text
待处理 -> 处理中 -> 已关闭
```

结案前检查配送商意见、附件、订单状态和账务影响。

## 5. 账务

- `/operation/billing-overview` 查看账务总览、账单和对账。
- `/operation/billing-cycles` 配置账期并生成账单。

生成账期前确认日期范围、主体范围和订单状态。定向账期与全局账期应避免重复覆盖。

## 6. 智能秤识别

入口：`/operation/smart-scale-recognition`

推荐顺序：

1. 建立识别类别。
2. 从智能秤收货图片导入样本或补录外部样本。
3. 审核类别并校准固定机位 ROI。
4. 重新裁剪受影响样本。
5. 创建训练任务。
6. 查看训练状态和指标。
7. 部署合格模型。
8. 发布终端使用的识别包。

保存 ROI 不会自动切换线上模型；需要重新训练、部署并发布。
