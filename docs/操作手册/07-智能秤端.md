# 智能秤端

项目：`/Users/Admin/Project/smart-scale-android`

## 1. 使用前准备

- 设备可访问后端 API。
- 首衡串口秤连接正确，默认设备和波特率以终端设置为准。
- UVC 摄像头已授权并能预览。
- 采购方账号已绑定至少一个启用食堂。

## 2. 登录与选食堂

1. 输入 API 根地址。
2. 使用采购方账号登录。
3. 选择当前收货食堂。
4. 进入待收货订单。

智能秤不能使用供货方或厂家账号收货。

## 3. 标准收货

1. 打开待收货订单。
2. 选择商品行。
3. 等待秤读数稳定。
4. 锁定重量；终端同时保存 UVC 商品照片。
5. 核对实收值。
6. 确认本行。
7. 完成全部行后，由仓库和承运方分别签字。
8. 提交整单收货。

## 4. 少收与退货

1. 在商品行填写实收值。
2. 选择少收或退货原因。
3. 填写退货数量。
4. 上传退货照片。
5. 提交后等待配送商复核（若该配送商启用了复核）。

不要用改订单应收数量的方式隐藏现场差异。

## 5. 演示模式

演示模式只用于没有后端或硬件的界面演示。演示数据不会形成正式订单、收货、退货、账单或识别样本。

## 6. 常见故障

- 无食堂：由运营端检查账号与食堂绑定。
- 无订单：检查所选食堂、订单状态和网络。
- 重量不变：检查串口设备、权限、波特率和数据帧。
- 无照片：检查 UVC 权限、占用和连接。
- 图片上传失败：检查 MinIO、API 根地址和网络。
