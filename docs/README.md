## 系统边界

- 六类数据库业务角色：`client`、`supplier`、`delivery`、`factory`、`operation`、`monitor`。
- 三个 Android 专用终端：智能秤、分检 PDA、司机端。
- 演示控制台、天枢、BI、图像识别属于辅助工具或模块，不是额外业务角色。
- 指定厂家使用 `factory` 角色，但业务本质是只承接指定厂家商品的特殊供货方。
- 当前部署基线是 Docker Compose；仓库内没有 Kubernetes 或 Helm 配置。

## 维护规则

1. 改业务状态、页面路由、接口路径或终端职责时，同一变更必须更新对应权威文档。
2. 不在 Markdown 中复制完整 OpenAPI；只记录业务入口和关键接口。
3. 不把计划、阶段记录、测试截图路径或临时隧道地址加入 RAG。
4. 技术内容先改 `系统技术文档.md`，再运行 `make docs-build` 生成 Word。
5. 提交前运行 `make docs-check`，确保文档没有引用已删除路由和接口。
6. 演示账号只允许写入内部演示指南，不写入正式技术或生产运维文档。

## 快速检查

```bash
make docs-build
make docs-check
```
