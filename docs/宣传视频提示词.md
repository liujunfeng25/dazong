```yaml
product_name: 大宗物资供应链全链路管理系统

business_roles:
  - 采购方
  - 供货方
  - 配送商
  - 指定厂家
  - 运营方
  - 监管方

  - 智能秤收货端

  - 司机端
supporting_modules:
  - 演示控制台
```

不得宣传以下不存在的功能：

- 工厂进厂、加工工单、生产排程和成品入库。
- 供货方独立库存系统。
- 已部署 Kubernetes。
- 无网络时仍能获得真实地图、实时行情或云端 AI 结果。
- 已弃用的 UniApp 智能秤端。

## 2. 视觉方向

- 风格：务实、可信、克制、现代工业。
- 主色：深蓝、白、工业灰。
- 点缀：橙色只用于提醒和确认。
- 场景：食堂后厨、仓库、配送中心、冷链车、运营办公室、监管大屏。
- UI 镜头应叠加真实系统截图，避免生成无法操作的假界面。
- 禁止赛博朋克霓虹、旋转粒子球、虚构数据大屏和错误中文。

## 3. 60 秒故事线

1. 清晨批发和供货场景。
2. 采购方选择食堂，通过商品检索、OCR 或语音辅助下单。
3. 配送商按报价和指定厂家规则智能分单。
4. 供货方和指定厂家打印标签并出库。
5. 配送中心使用 PDA 扫码分检。
6. 调度员生成路线、装车并发车。
7. 司机查看车次、地图和站点，确认送达。
8. 食堂使用智能秤称重、拍照、退货留痕和双签收货。
9. 账单与对账生成，监管端查看履约、价格、风险和审计。
10. 收尾文案：“六类角色，三个专用终端，一张供应链数据网。”
